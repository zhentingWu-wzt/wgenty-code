Subagent Panel — 错误展示与恢复交互

当 subagent 执行失败时，panel 中如何展示错误信息并提供恢复操作？

A

内联展开（推荐）

选中 Failed 节点后，内联展开错误详情 + 操作按钮。不离开 panel 上下文，操作直观。

🌳 Subagent Tree
├─ ✅ explore skills — 14 rounds · 66.2s
├─ ✅ explore plugins — 16 rounds · 94.0s
├─ ❌ sub: fix auth bug — Failed
Error: Subagent timed out after 240s
Last tool: file_read("src/auth/login.rs")
Tokens: 12.5k/50k · Round: 8/20
[r] retry [d] details [Esc] close
B

Popup 详情框

错误信息以浮层弹窗展示，覆盖在 panel 上方。信息密度更高，但打断 panel 导航流。

🌳 Subagent Tree
├─ ✅ explore skills
├─ ❌ sub: fix auth bug — Failed
├─ ✅ explore plugins
❌ Subagent Failed
Timeout after 240s
Last: file_read("src/auth/login.rs")
12.5k/50k tokens · round 8/20
[r] retry [d] details [x] dismiss
C

状态栏 + 节点标记（最小模式）

错误摘要显示在状态栏，节点只标注 Failed 状态。用户需按 Enter 进入 detail view 查看完整信息。最简洁但信息延迟。

NORMAL | ⚠ 1 failed | 2/3 done · 14.2s
🌳 Subagent Tree
├─ ✅ explore skills
├─ ❌ sub: fix auth bug — Failed [Enter: details]
├─ ✅ explore plugins
