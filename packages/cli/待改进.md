# Wgenty Code — 待改进能力清单

> 对照 Claude Code / Codex / Copilot 等成熟 coding agent 分析差距。

---

## 一、上下文理解 & 项目感知

1. **Repo Map / 代码库索引** — 构建 AST 符号图，理解全仓库结构后再行动，而非仅依赖 grep 文本搜索。
2. **项目规则文件（CLAUDE.md / `.claude/`）** — 用户在项目根目录放置编码规范、架构说明，agent 启动时自动读取并遵守。
3. **跨会话记忆** — 记住用户偏好、常用命令、项目约定，持久化到本地。
4. **Git 深度上下文** — 自动注入最近 commits、未暂存变更、diff、blame 等作为决策参考。

---

## 二、Agent 规划 & 推理能力

5. **显式 Plan模式** — 先列出 todo 计划再逐步执行，用户可以审查和调整计划后再动手（当前 `task_management` 工具已有但缺乏规划工作流）。
6. **推理链可见性** — `onReasoningDelta` 目前被丢弃，应支持可选展示（类似 Claude Code 的 thinking mode）。
7. **Self-correction 循环** — 工具结果不符合预期时，自动分析原因、调整策略、重试。
8. **并行工具执行** — 当前全部串行，多个独立工具调用（如同时读两个文件）应并行发起。

---

## 三、工具扩展 & 生态

9. **MCP（Model Context Protocol）支持** — 让用户接入自定义工具服务器，是 Claude Code 最重要的差异化能力。
10. **LSP 集成** — 定义跳转、引用查找、诊断信息，实现语义级代码理解。
11. **Web 搜索 & URL 抓取** — 查询最新文档、API 参考、GitHub issues。
12. **Diff Preview** — 编辑文件前展示将要修改的内容，用户确认后再 apply（当前直接写入）。
13. **实时命令输出流** — 长命令（如 `npm install`、`cargo build`）逐步显示输出，而非等结束一次性返回。

---

## 四、安全 & 权限

14. **细粒度权限规则** — 按路径（`/src/**` 允许写、`/etc/**` 拒绝）、命令模式（允许 `npm test`、禁止 `rm -rf`）设置白名单/黑名单。
15. **沙箱执行** — 高风险命令在容器或隔离环境中运行。
16. **审计日志** — 记录所有文件变更历史，可追溯可回滚。
17. **Token 用量与成本追踪** — 实时统计消耗、设置预算上限。

---

## 五、用户体验

18. **会话管理** — 保存/恢复/命名会话、会话列表、搜索历史会话。
19. **Undo 机制** — 撤销最近的文件修改或工具操作。
20. **输入补全** — 文件路径、命令、项目符号的自动补全。
21. **down 渲染** — assistant 消息当前为纯文本，不支持代码块语法高亮、表格、列表等格式。
22. **历史搜索** — `Ctrl+R` 搜索历史输入。
23. **自定义主题** — 用户可配置颜色和样式。

---

## 六、多模型 & 弹性

24. **多 Provider 切换** — 支持 OpenAI / Anthropic / 本地模型，当前强依赖单一 Rust daemon。
25. **模型路由** — 简单问题用便宜模型，复杂问题自动用强大模型。
26. **Fallback 机制** — 一个模型出错时自动切换备选。
27. **本地/离线模式** — 不依赖远端 daemon 的独立运行能力。

---

## 七、调试 & 可观测性

28. **Verbose / Debug 模式** — 显示完整 system prompt、API 请求/响应供调试。
29. **Replay 模式** — 重放历史会话用于复现和回归测试。

---

## 八、代码架构改进（当前即可着手）

| # | 问题 | 建议 |
|---|------|------|
| 30 | 工具调用全部串行 | 实现并行执行调度器，无依赖的工具同时发起 |
| 31 | 无重试/超时机制 | 网络抖动或工具卡死直接失败，需加入指数退避重试与超时控制 |
| 32 | 推理内容直接丢弃 | `onReasoningDelta` 应可选展示，支持 thinking mode |
| 33 | 会话退出即丢失 | 需实现对话持久化（SQLite / JSON 文件） |
| 34 | `@claude-code/core` 耦合度高 | 抽象 AgentLoop 接口，支持切换后端 |
| 35 | 工具集硬编码在多处 | `render.ts`、`use-agent.ts` 各有一份工具格式化代码，应统一到单一 registry |
| 36 | 流式渲染 throttle 固定 100ms | 可改为自适应策略，内容少时更快刷新 |

## 优先级建议

| 优先级 | 能力 | 理由 |
--------|------|------|
| 🔴 P0 | MCP 工具扩展 | 生态开放性，决定 agent 天花板 |
| 🔴0 | Repo Map / 代码索引 | 从"文本搜索"到"语义理解"的质变 |
| 🔴 P0 | Plan 模式 + 推理可见 | agent 从"执行命令"到"理解→规划→执行→验证" |
| 🟡 P1 | LSP 集成 | 精准的定义/引用/诊断 |
| 🟡 P1 | 会话持久化 + Undo | 基础用户体验 |
| 🟡 P1 | 细粒度权限 + 审计 | 生产可用性 |
| 🟢 P2 | 多 Provider / 模型路由 | 灵活性和成本优化 |
| 🟢 P2 | Markdown 渲染 / 补全 | UI 打磨 |
| 🟢 P2 | Debug / Replay 模式 | 开发者体验 |
