4. 从 Session JSON 批量统计 codegraph 使用率的可行性

   可行。

   所有 71 个 session JSON 均包含 "messages" 字段（schema 一致率 100%），
   因此可以直接用 jq 批量解析。关键 jq 查询模式：

   - 检查是否有 codegraph 调用:
     jq '[.messages[]?.tool_calls[]? | select(.function.name | test("codegraph"))] | length'

   - 统计所有工具调用:
     jq '[.messages[]?.tool_calls[]?] | length'

   注意事项:
   - jq 的 "?." 操作符（optional chaining）对缺少 messages 或
     tool_calls 字段的 session 安全，返回 null/空数组而非报错
   - 所有 session 均有 messages 字段（本次统计未发现异常）
   - 统计结果可以通过 SESSIONS_DIR 环境变量覆盖，便于测试

5. 需要注意的陷阱

   a) Schema 一致性: 本次扫描的 71 个 session 全部包含 messages 字段，
      未发现 schema 不一致。但未来可能出现旧格式 session，建议在脚本中
      使用 jq 'has("messages")' 做防御性检查。

   b) tool_calls 可选性: 即使有 messages 字段，也可能不存在 tool_calls
      （如纯对话 session）。jq 的 "[]?" 操作符能安全处理此情况。

   c) 工具命名规范: codegraph 工具目前仅有 codegraph_node 被实际使用。
      搜索时使用 test("codegraph") 正则匹配可覆盖 codegraph_node、
      codegraph_explore 及未来 codegraph_* 工具。

   d) Bash 兼容性: macOS 默认 bash 3.2 不支持关联数组 (declare -A)，
      管道 + sort | uniq -c 是更安全的统计模式。

   e) Token 用量: Session JSON 不存储 API 返回的 usage(token 用量)信息，
      无法从 session JSON 获取 codegraph 调用的 token 成本。
