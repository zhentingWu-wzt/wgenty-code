现在我对这个项目有了全面的了解。这是一个模仿 **Wgenty Code** 的 CLI 编程智能体——「Wgenty Code · Rust            │
│ Edition」，采用 Rust daemon + Node.js/Ink TUI 前端架构。                                                        │
│                                                                                                                 │
│ 让我对照目前业界最强的两个 coding agent——**Wgenty Code 和 **OpenAI Codex CLI**——来分析缺失的核心能力：          │
│                                                                                                                 │
│ ---                                                                                                             │
│                                                                                                                 │
│ ## 🔴 一、工具 & 集成层                                                                                         │
│                                                                                                                 │
│ ### 1. **MCP (Model Context Protocol) 支持**                                                                    │
│  Wgenty Code 最大的差异化能力。你的项目目前只有硬编码的 10 个工具，完全没有 MCP client——无法连接任何外部 MCP    │
│ server（如 Postgres、SlackJira、自定义工具等）。这意味着工具生态完全封闭。                                      │
│                                                                                                                 │
│ ### 2. **LSP (Language Server Protocol) 集成**                                                                  │
│ 虽然你在 `render.ts` 中有一个 `lsp` 的 icon 映射，但实际上**没有 LSP client 实现**。真正的 Wgenty Code 会利用   │
│ LSP 做：                                                                                                        │
│ - 跳定义 (go-to-definition)                                                                                     │
│ - 查找引用 (find references)                                                                                    │
│ - 诊断信息 (diagnostics/warnings/errors)                                                                        │
│ - 代码补全感知                                                                                                  │
│                                                                                                                 │
│ ### 3. **Web Search / RAG 检索**                                                                                │
│ 无法联网搜索文档、GitHub issues、Stack Overflow 等外部上下文。Codex 和 Wgenty Code 都能做实时搜索。             │
│                                                                                                                 │
│ ### 4. **Image / 截图粘贴支持**                                                                                 │
│ Wgenty Code 支持直接在终端粘贴截图作为上下文（通过 iTerm2 的 inline image                                       │
│ 或文件路径）。你的项目完全没有这个能力。                                                                        │
│                                                                                                                 │
│ ---                                                                                                             │
│                                                                                                                 │
│ ## 🔴 二、Agent 架构层                                                                                          │
│                                                                                                                 │
│ ### 5. **Sub-agent / 并行委派**                                                                                 │
│ Wgenty Code 可以 spawn 多个子 agent 并行处理不同任务（比如同时改 3 个文件），然后汇总结果。你目前是单线程       │
│ AgentLoop。                                                                                                     │
│                                                                                                                 │
│ ### 6. **Plan Mode（规划模式）**                                                                                │
│ 在真正写代码之前先进行架构讨论、生成计划、让用户确认。Codex 和 Wgenty Code 都有显式的「先想再做」模式。你只有   │
│ thinking → execute 的简单流程。                                                                                 │
│                                                                                                                 │
│ ### 7. **Testing Loop（自动测试修复循环）**                                                                     │
│ Wgenty Code 的核心体验：改代码 → 跑测试 → 看失败 → 自动修复 → 再跑测试 → 循环直到通过。你目前没有这个能力。     │
│                                                                                                                 │
│ ### 8. **Background Tasks（后台任务）**                                                                         │
│  `npm run dev` 或 `cargo build` 这种长时任务时，能放到后台执行，同时继续对话。你的是同步阻塞的。                │
│                                                                                                                 │
│ ---                                                                                                             │
│                                                                                                                 │
│ ## 🔴 三、上下文 & 记忆层                                                                                       │
│                                                                                                                 │
│ ### 9. **CLAUDE.md / 项目级 Rules 系统**                                                                        │
│ Wgenty Code 会读取项目根目录的 `CLAUDE.md`（或                                                                  │
│ `.claude/rules/`）作为持久化的项目规则和上下文。你的项目完全没有。                                              │
│                                                                                                                 │
│ ### 10. **Memory / 跨会话记忆**                                                                                 │
│ 在对话中学到的偏好、项目约定、用户习惯等，跨会话持久化。你每次都是全新 agent。                                  │
│                                                                                                                 │
│ ### 11. **Auto-compact / 上下文自动压缩**                                                                       │
│ 当 conversation history 太长时，自动总结前面的内容并压缩。你目前只会无限增长，很快就会爆 context window。       │
│                                                                                                                 │
│ ### 12. **智能 Context 管理**                                                                                   │
│ 根据当前任务自动选择相关文件放进 context，不需要用户显式 `@file` 。Wgenty Code 会分析项目结构、git history      │
│ 等自动收集上下文。                                                                                              │
│                                                                                                                 │
│ ---                                                                                                             │
│                                                                                                                 │
│ ## 🔴 四、用户体验层                                                                                            │
│                                                                                                                 │
│ ### . **Rich Diff 展示**                                                                                        │
│ Wgenty Code 的文件编辑会展示 **syntax-highlighted unified diff**（类似 `git diff`），而不仅仅是文本替换。你的   │
│ `file_edit` 工具没有 diff 预览。                                                                                │
│                                                                                                                 │
│ ### 14. **Custom Slash Commands**                                                                               │
│ 用户可以定义 `/deploy`、`/review`、`/test` 等自定义命令。你现在只有硬编码的                                     │
│ `exit`、`clear`、`reset`、`history`、`help`。                                                                   │
│                                                                                                                 │
│ ### 15. **val Modes / 权限分级**                                                                                │
│ Wgenty Code 有多种权限模式：                                                                                    │
│ -Auto-approve**: 自动批准所有操作                                                                               │
│ - **Approval required**: 每次需要确认                                                                           │
│ - **Per-tool policy**: 对不同工具设置不同策略（如读文件自动批准，执行命令需确认）                               │
│ 你目前只有简单的 `[y] Allow once [a] Alwaysn] Deny`。                                                           │
│                                                                                                                 │
│ ### 16. **Checkpoint / Undo**                                                                                   │
│ 执行完一组操作后，可以回退到之前的状态。Wgenty Code 通过 git 实现 checkpoint。你没有。                          │
│                                                                                                                 │
│ ### 17. **Hook 系统**                                                                                           │
│ Pre/post tool hooks——比如在 `file_write` 之前自动 run linter，在 `execute_command` 之后自动检查输出。没有。     │
│                                                                                                                 │
│ ---                                                                                                             │
│                                                                                                                 │
│ ## 🔴 五、基础设施                                                                                              │
│                                                                                                                 │
│ ### 18. **CLI Argument Parsing / One-shot Mode**                                                                │
│ `claude "fix the bug in auth.ts"` 一次性执行然后退出。你虽然支持 initial prompt，但没有用正经的 CLI argument    │
│ parser（如 `yargs`/`commander`），也没有 `--model`、`--`、`---run` 等标准 CLI flags。                           │
│                                                                                                                 │
│ ### 19. **Session Resume**                                                                                      │
│ 中断后可以用 `claude --resume` 恢复上次的会话。不支持。                                                         │
│                                                                                                                 │
│ ### 20. **Streaming Tool Output**                                                                               │
│ `execute_command` 的结果应该是**实时流式输出**的，而不是等命令结束才一次性返回。这对长时编译任务至关重要。      │
│                                                                                                                 │
│ ### 21. **Multi-turn Tool Orchestration**                                                                       │
│ 真正复杂场景下 agent 需要连续调用 20+ 个工具，可能交错进行（读 A → 读 B → 改 → 执行命令 → 读结果 → 改           │
│ B）。你目前的 AgentLoop 是否支持这种复杂编排，从代码中看不出来，但如果是从 `@wgenty-code/core`                  │
│ 继承的，可能有一定能力。                                                                                        │
│                                                                                                                 │
